<h1 align="center">
  ワンタッチ撮影ツール (CameraAutomationTool)
</h1>
<p align="center">
  投薬前に医薬品全体を撮影する業務の自動化ツール。医薬品提供後に数や種類が合わないといったクレームを防止する。
</p>
<br/>

## 使用言語
- PowerShell + .NET Framework + Win32API
- C# (スクリプト実行用のランチャー部分のみ)

## 開発背景
　弊社の複数のグループ店において **「投薬後の医薬品の数や種類が合わない！」** といったクレームが患者様から寄せられる事案が定期的に報告されている。実際には正しく渡しているケースがほとんどであるが、証拠をすぐに確認できないため泣き寝入りになることすらある状況であった。そこで投薬前に医薬品の写真を撮るという解決策が提案され、実証実験店舗には撮影用にWEBカメラが配布された。

　撮影を手作業で行う場合、【カメラアプリの起動】➔【撮影ボタンの押下】➔【カメラアプリの終了または最小化】という手間がかかり、さらには撮影後の画像の整理も必要。薬歴記載のために、ブラウザで薬歴アプリを全画面表示している環境ではとても面倒な作業であった。薬剤師への聞き取りでも、 **操作自体が面倒でやらなくなってしまったり、後回しにしたことで忘れてしまったりということが多い** とのことであった。

　本社システム課より、この撮影業務を楽にすることはできないかとの提案を受け **【撮影動作のワンボタン化】** **【画像保存作業の自動化】** が技術的に可能と判断し、本ツールの開発を行うこととなった。

## 開発概要
>[!NOTE]
>本アプリケーションの実行環境である投薬・薬歴記載用PCは、セキュリティが厳しく設定され、外部ツールの導入および外部との通信が著しく制限されていることから、Windows標準の機能であるPowerShellおよび.NET組み込みのC#コンパイラを利用する形とした。

　本ツールで自動化するのは **「カメラアプリの起動から撮影、アプリ終了までの手順」** と **「指定したパスおよびタイトルでの画像保存」** である。画面上に撮影ボタンを常駐させるUIを実装し、使用者は画面上のボタンを任意のタイミングで押下するだけで、撮影位置に用意した医薬品の画像を撮影できる。また、撮影のタイミングでは薬品を渡す患者様の薬歴画面を開いていることを利用し、 **薬歴アプリ(Google Chromeブラウザ)から患者名を取得し、撮影日付・時間とともに画像ファイル名に付加する機能** も持たせた。オプション画面にて、保存先のパス指定や患者名保存の有効/無効切り替えも可能。

　スクリプト自体はC#で設計したシンプルなランチャーから実行する形にすることで、アイコン設定、タスクバーへの登録ができるようにし、スクリプトであることを意識せずに使えるようにした。

## 実績
　システム課より提案いただいてから2週間程度でプロトタイプを開発、自店舗にて実証実験を開始。1か月ほどの試用期間の最中に、薬剤師よりフィードバックをいただき、保存先の指定や患者名の付加機能を追加。また、他店舗への横展開を想定して、PC環境によらずに動作するようにハードコーディング部分を減らし、ウェイト時間等をオプションで指定できるように調整した。

　これらの追加機能も好評をいただき、試用期間終了後、システム課の上長からの指示により、現在は数店舗への段階的な横展開でさらなる実証実験を進めている。展開から1か月ほどが経過しているが、現時点では使用方法についての問い合わせがあったのみで、エラー報告等はなく動作している。

## 確認済み動作環境
- Windows 10/11
- PowerShell ver.5.0 以上
- .NET Framework 4.x 導入済み

## インストール方法
本リポジトリの内のファイルをすべて一つのフォルダに入れ、任意の場所に保存するだけでOK。

>[!TIP]
>インストールオプションとして、タスクバーにピン止めするためのアシスタントを同梱。  
>**【ワンタッチ撮影をタスクバーに登録.bat】** を実行すると、ランチャー経由でスクリプトが起動する。その後、タスクバーにピン止めする旨のメッセージが表示されるので、その間にタスクバーへピン止め。  
>本来なら、タスクバーに登録できないPowerShellスクリプトをランチャー経由で起動し、一時停止させることで、タスクバーに実行ファイルとして登録させる工夫となっている。

## 使用方法
1. ワンタッチ撮影.exeからスクリプトを起動。
2. 画面上に最前面で常駐する **【撮影ボタン】** が表示されるので、撮影したいタイミングで押下。
3. 右クリックメニューからオプション画面を表示可能。
<br/>

オプションでは、以下の項目が設定可能。
- 保存先のパス
- 撮影前の待機時間（PCの環境ごとにカメラアプリの起動時間に合わせて調整）
- 撮影後の待機時間（PCの環境ごとにカメラアプリの撮影後処理に合わせて調整）
- リトライ回数（カメラアプリの起動タイミングがずれてしまった場合用に、撮影ボタンを検出するリトライ回数を調整）
- 画像を自動で削除する期間の指定
- 画面上に常駐ボタンを表示するか、アプリ起動で直接撮影するかの設定
- 一回の撮影ごとにカメラアプリを閉じるかどうか
- 一回の撮影ごとにカメラアプリを最小化するかどうか
- 薬歴アプリで開いている患者名をファイル名に付加するかどうか

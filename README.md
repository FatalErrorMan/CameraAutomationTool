# CameraAutomationTool（ワンタッチ撮影ツール）
医薬品投薬後の個数・種類が違っているというクレームを防止するため、投薬前に医薬品全体を撮影する業務を自動化するツール。

## 開発背景
弊社の複数のグループ店において、投薬後の医薬品の個数や種類が違うといったクレームが患者様から寄せられる事例が定期的に報告されている。  
実際には正しく渡しているケースがほとんどであるが、その証拠を確認できないために泣き寝入りになることもあり、投薬前に医薬品の写真を撮るという解決策が提案された。実証実験店舗には、撮影用にWEBカメラが配布され、投薬・薬歴記載用PCに接続されている。

手作業で行う場合、カメラアプリの起動->撮影ボタンの押下->カメラアプリの終了または最小化の手順が必要であり、さらには撮影後の画像の整理もしなければならなかった。薬歴記載用にブラウザで薬歴アプリを全画面に開いている環境では面倒な作業であり、薬剤師への聞き取りでも、 **操作自体が面倒でやらなくなってしまったり、後回しにしたことで忘れてしまったりということが多い** とのことであった。

本社システム課より、この撮影業務を楽にすることはできないかとの提案を受け、 **撮影動作のワンボタン化** 、**画像保存作業の自動化** が技術的に可能と判断し、本ツールの開発を行うこととなった。

## 使用言語
PowerShell, C#(スクリプト実行用のランチャー部分のみ)

## 開発概要
アプリケーションの実行環境である投薬・薬歴記載用PCはセキュリティが厳しく設定され、外部ツールの導入および外部との通信が著しく制限されていることから、Windows標準の機能であるPowerShellおよび.NET組み込みのC#コンパイラを利用する形とした。

本ツールで自動化するのは **「カメラアプリの起動から撮影、アプリ終了までの手順」** と **「指定したパスおよびタイトルでの画像保存」** である。画面上に撮影ボタンを常駐させるUIを実装し、使用者は画面上のボタンを任意のタイミングで押下するだけで、撮影位置に用意した医薬品の画像を撮影できる。また、撮影のタイミングでは薬品を渡す患者様の薬歴画面を開いていることを利用し、 **薬歴アプリ(Google Chromeブラウザ)から患者名を取得し、撮影日付・時間とともに画像ファイル名に付加する機能** も持たせた。

オプション画面にて、保存先のパス指定や患者名保存の有効/無効を切り替えも可能。

スクリプト自体はC#で設計したシンプルなランチャーから実行する形にすることで、アイコン設定、タスクバーへの登録ができるようにし、スクリプトであることを意識せずに使えるようにした。

## 実績
システム課からの提案から2週間程度でプロトタイプを開発、自店舗にて実証実験を開始。1か月ほどの試用期間の最中に、薬剤師よりフィードバックをいただき、保存先の指定や患者名の付加機能を追加。また、他店舗への横展開を想定して、PC環境によらずに動作するようにハードコーディング部分を減らし、ウェイト時間等をオプションで指定できるように調整した。

これらの追加機能も好評をいただき、試用期間終了後、システム課の上長からの指示により、現在は数店舗への段階的な横展開でさらなる実証実験を進めている。展開から1か月ほどが経過しているが、現時点では使用方法についての問い合わせがあったのみで、エラー報告等はなく動作している。

## 導入方法・操作方法
①本リポジトリの内のファイルをすべてフォルダに入れ、任意の場所に保存。

オプション：【付属のワンタッチ撮影をタスクバーに登録】と書かれたショートカットをダブルクリック。  
※本リポジトリにはアップロードしていない。内部動作としては、ランチャーに対して、スクリプト起動時にポーズするためのオプションを指定し、ポーズ中にタスクバーにピン止めする旨のメッセージを表示する。

②ワンタッチ撮影.exe(ランチャー)からスクリプトを起動。

③画面上に最前面で常駐するボタンが表示されるので、撮影したいタイミングで押下。

④右クリックメニューからオプション画面を表示可能。  
オプションでは、以下の項目が設定可能。
+ 保存先のパス
+ 撮影前の待機時間（PCの環境ごとにカメラアプリの起動時間に合わせて調整）
+ 撮影後の待機時間（PCの環境ごとにカメラアプリの撮影後処理に合わせて調整）
+ リトライ回数（カメラアプリの起動タイミングがずれてしまった場合用に、撮影ボタンを検出するリトライ回数を調整）
+ 画像を自動で削除する期間の指定
+ 画面上に常駐ボタンを表示するか、アプリ起動で直接撮影するかの設定
+ 一回の撮影ごとにカメラアプリを閉じるかどうか
+ 一回の撮影ごとにカメラアプリを最小化するかどうか
+ 薬歴アプリで開いている患者名をファイル名に付加するかどうか
